가이드 안내

퍼블로그 디자인 시스템의 시각적 레퍼런스 페이지예요. AI가 아닌 사용자가 직접 확인하는 용도예요.
디자인 시스템은 클로드 툴을 통해 사용할 수 있어요.
클로드 앱으로 시작하기
claude.ai에서 스킬을 활성화해 HTML을 생성해요
클로드 디자인으로 시작하기
claude.ai/design에서 디자인 시스템을 선택해 작업해요

이렇게 사용해주세요

완성 후 웹팀에 검수를 요청해주세요
AI는 가이드를 항상 완벽하게 따르지 않아요. 가이드에 없는 요소는 임의로 생성하기도 해요.
작업이 끝나면 노션 양식으로 웹팀에 검수 요청해 주세요.
대화창(세션)을 최적화해 관리해주세요
대화가 길어지면 Claude가 이전 내용을 자동으로 압축(컴팩트)해요.
이 과정에서 작업 내용이 소실되거나 가이드 참조가 약해질 수 있어요.
대화창에 내용이 길어지면 HTML을 내 컴퓨터에 저장해두고, 새 대화창에 파일을 첨부해 이어 작업하세요.

클로드 앱으로 시작하기

클로드 앱 스킬을 사용해 퍼블로그 스타일 HTML을 만드는 방법이에요. Chat 또는 Cowork 모두 사용 가능해요.

사용 방법

1
채팅창 진입하여 스킬 시작
방법 A 채팅창 하단 "+" → 스킬 목록에서 'publog-design-system' 선택
방법 B 채팅창에 "퍼블로그 디자인 시스템" 입력 → 트리거로 자동 활성화
2
원하는 페이지·레이아웃 디자인 작업
3
HTML 결과 확인 → 수정 반복
4
채팅창에 "작업끝" 입력
HTML 파일이 제공되고, 웹팀 확인 요청 마무리 단계로 진입해요.
5
노션에 업로드 → 웹팀에 확인 요청

퍼블로그 웹 디자인 표준화 시스템을 관리하는 담당자를 위한 설명이에요.

6
노션에서 "작업 완료" 처리
요청자에게 완료 내역 공유하기
GitHub가 뭐예요?
파일을 인터넷에 저장·배포하는 클라우드 저장소예요.
Claude가 이 저장소에서 디자인 가이드를 자동으로 불러와 항상 최신 상태를 유지해요.
연동은 이미 완료되어 있어서 별도 세팅 없이 파일 수정 후 Claude Code에 작업을 맡기면 자동으로 처리돼요.

클로드 코드로 작업하기

처음이라면 이렇게 시작하세요
클로드 코드로 프로젝트가 자동화되어 있어요. 관리자는 AI가 안내하는 흐름에 따라 작업하면 돼요.
VS Code를 프로젝트 경로에 연결하고 채팅창에 새 작업이라고 입력해보세요.

주요 작업 명령어

디자인 변경은 명령어 없이 바로 말하면 돼요. 예: "배너 추가해줘", "버튼 색상 바꿔줘"
명령어설명
새 작업노션 확인 → received/ 파일 준비 → 검수 → 수정까지 단계별 안내해요.
GitHub 업로드·로그·노션 완료는 마무리 단계에서 자동 처리돼요.
이어서LOG.md 마지막 항목과 노션 미완료 항목을 확인해 이전 작업을 어느 단계부터 재개할지 안내해줘요.
백업해줘버전 확정 + zip + 가이드 페이지 스냅샷을 backup/ 폴더에 저장해요.
로그 저장대화를 중간에 끊을 때 작업 내용을 LOG.md에 바로 기록해요.
로그 저장을 잊지 마세요
클로드 코드에서 대화가 가득 차면 이전 내용이 자동 압축(compact)되면서 작업 맥락을 잃을 수 있어요.
세션이 끝나기 전에 반드시 로그 저장을 실행해주세요.
LOG.md에 기록이 남으면 다른 관리자도 새 세션에서 이어서로 작업을 이어받을 수 있어요.
